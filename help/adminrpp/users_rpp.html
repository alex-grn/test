2.2.2 Пользователи

Подраздел «Пользователи» предназначен для регистрации пользователей системы, определения им уровня доступа.

Подраздел разделен на две экранные формы (Рисунок 7). В главной экранной форме (Главная таблица) отображаются данные по аккаунтам пользователей, а в нижней экранной форме (Таблица спецификации) осуществляется определение ролей для каждого аккаунта.

 

Рисунок 7 - Экранная форма раздела «Пользователи»

 

Для добавления нового пользователя в главной таблице вызвать контекстное меню и выбрать «Добавить». Откроется форма (Рисунок 8). В открывшейся форме заполнить следующие поля:

-Логин (Краткое имя сотрудника (Фамилия и инициалы));

-Пароль – идентификации для входа в веб-сервис;

Наименование – краткое описание аккаунта пользователя;

Уровень доступа по умолчанию – присвоение прав доступа.

Нажать «ОК».

 

Рисунок 8 - Форма создания нового пользователя

 

Для определения прав доступа для каждого пользователя в таблице спецификации вызвать контекстное меню и выбрать «Добавить». В открывшейся форме выбрать необходимую роль из списка (Рисунок 9).

 

Рисунок 9 - Назначение ролей пользователям

 

При необходимости расширения уровня доступа, пользователю необходимо назначить несколько ролей.

Для блокировки пользователя вызвать контекстное меню и выбрать «Исправить». В открывшейся форме активировать флаг «Запись заблокирована», нажать «ОК». При попытке зайти в Систему, заблокированному пользователю Система выдаст сообщение (Рисунок 10).

 

 

Рисунок 10 - Вход в систему не возможен

 

Для удаления, ранее зарегистрированного пользователя в главной таблице вызвать контекстное меню и выбрать «Удалить». После подтверждения намеренности действий (Рисунок 11) – запись о пользователе будет удалена безвозвратно.

 

Рисунок 11 - Форма подтверждения удаления пользователя

 

Важно! Удаление пользователей не возможно, в случае если предполагаемый для удаления аккаунт, был ранее задействован в рабочем процессе. В этом случае использовать только «Заблокировать пользователя».
